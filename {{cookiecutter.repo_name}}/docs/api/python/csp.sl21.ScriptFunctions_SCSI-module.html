LogSense(pg_code, sub_pg_code=0, alloc_len=0xffff, pc=1, sp=1, param_ptr=0)

source code  

Issue Log Sense command to current device.

pg_code: integer:     Log page code
sub_pg_code: integer: Log page sub code
alloc_len: integer:   Amount of log page data to read. (Default 0xffff)
pc: integer:          Page control - type of log parameter values to return (Default 1)
                          0: threshold values
                          1: cumulative values
                          2: default threshold values
                          3: default cumulative values
sp: integer:          Save parameters (Default 1)
                          0: LogSense will not cause any parameters to be saved
                          1: LogSense will attempt to save parameters. Note this will
                             cause illegal request if device does not support saving of
                             parameters via Log Sense command.
param_ptr: integer:   Parameter pointer field (Default 0). Allows user to read partial
                      log pages by specifing a parameter code. Log Sense will use the
                      parameter code and skip all parameter codes that come before the
                      specified parameter. Will fail for invalid field in CDB if
                      given parameter code is larger than largest available parameter code.
Decorators:

  • @misc.restorePluginParameters

LogSelect(pg_code, sub_pg_code=0, param_list_len=0xffff, pc=1, sp=0, pcr=0)

source code  

Issue Log Select command to current device.

pg_code: integer:         Log page code
sub_pg_code: integer:     Log page sub code (Default = 0)
param_list_len: integer:  Amount of log page data to send. (Default 0xffff)
pc: integer:              Page control - type of log parameter values to return (Default = 1)
                              0: threshold values
                              1: cumulative values
                              2: default threshold values
                              3: default cumulative values
sp: integer:              Save parameters (default 0)
                              0: LogSelect will not cause any parameters to be saved
                              1: LogSelect will update and save log parameters.
pcr: integer:             Parameter Code Reset - if set to one with param_list_len = 0,
                          it will cause this log page to be reset to default values.
                          If pcr = 1 and param_list_len != 0 will be terminated with
                          check condition status. (Default = 0)

ReadDefectData10(plistv, glistv, defect_list_format, alloc_len)

source code  

Issue Read Defect Data (10) command to pull defect list from the device.

plistv: boolean:             Request primary defect list to be returned
                             (Not to be confused with pending list on ATA devices)
glistv: boolean:             Request grown defect list to be returned
defect_list_format: integer: Specifies format to be used in returned defects. Uses same
                             address descriptor used by FORMAT UNIT command.
                             0: use short block format (LBA range upto 0xFFFF FFFF)
                             3: use long block format (LBA range upto 0xFFFF FFFF FFFF FFFF)
                             4: use bytes from index (Cylinder, head and bytes from index)
                             5: use physical sector format (Cylinder, head and sector)
alloc_len: integer           Amount of data to read from device (maximum 0xFFFF)

LogPage_BER(lpReset=1, headNum=-1, doDITSunlock=False)

source code  

Collects BER log page parameters from log page 0x03 (read) (parameters 0x0003, 0x0005, 0xE400),
calculates drive-level BER and sends it to database.

lpReset;boolean: Whether to reset log page after BER data is pulled or not. (Default = 1)
                 NOTE - if device rejects resetting log page 0x03, it will issue
                 a command to reset ALL log pages.
headNum;integer: Head number the calculated BER relates to. Default is -1 which is for all heads.

ModeSelectStrict()

source code 

Zeroes out first and third bytes of Mode Select bit, and also PS (pave saveable bit) for strict bit compliance. Returns size of the total mode select buffer length (including mode page contents and header) NOTE - Must be called after Mode Sense command, but before any manual manipulation of bits. Issue Mode Select using the mode page length returned by this function.
